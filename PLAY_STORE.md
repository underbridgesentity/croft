# Croft on Google Play - runbook

The Android app is a **Trusted Web Activity (TWA)**: the Play-installed app is
real Chrome rendering www.croftapp.co.za full-screen. Web deploys reach it
instantly (like iOS); **push notifications use the existing Web Push channel**
- no Firebase, no new server code.

Everything below `android/` is generated by Bubblewrap from `twa-manifest.json`
(JDK at ~/.bubblewrap, SDK at ~/Library/Android/sdk - both installed).

## One-time, Joseph only (secrets stay with you)

```sh
cd android
CROFT_KEYSTORE_PASS='choose-a-strong-password' ./create-keystore.sh
CROFT_KEYSTORE_PASS='...' ./gen-assetlinks.sh   # writes web/public/.well-known/assetlinks.json
CROFT_KEYSTORE_PASS='...' ./build.sh            # -> app-release-bundle.aab
```

Back up `android/android.keystore` + its password (password manager). Then
commit + deploy the assetlinks file (ask Claude or: git add web/public/.well-known
&& git commit && git push).

## Play Console (once)

1. https://play.google.com/console/signup -> **Organization** account
   (Under Bridges Entity (Pty) Ltd, D-U-N-S **654400173**), $25 once.
   Organization accounts skip the 20-tester/14-day requirement personal
   accounts have.
2. Create app: "Croft - Family Hub", App (not game), Free.
3. **Test and release -> Internal testing** -> upload `app-release-bundle.aab`
   (accept Play App Signing). Install on a real Android phone and check the
   app opens FULL-SCREEN (no browser bar). If a bar shows, assetlinks isn't
   matching yet - see next step.
4. **Setup -> App signing**: copy the **App signing key certificate SHA-256**
   and add it as a second entry in `web/public/.well-known/assetlinks.json`
   (keep the upload-key one), deploy. This is required - Play re-signs the
   app with Google's key.
5. **Store listing**: copy from `android/store-assets/listing.md`; upload
   `feature-graphic.png` + the five `0*.png` screenshots + the 512 icon
   (web/public/icons/icon-512.png).
6. **Data safety** answers:
   - Collects: Email address + Name (Account management), User-generated
     content / app activity (App functionality). Nothing else.
   - All data encrypted in transit: yes. Users can request deletion: yes
     (in-app: Family -> Account & security -> Delete account, or /support).
   - Shared with third parties: no. Ads: no.
7. **Content rating** (IARC): fill honestly -> Everyone. **Target audience**:
   18+ (Croft is not designed for children - avoids the Families program).
8. **App content**: privacy policy https://www.croftapp.co.za/privacy;
   app is free, no ads, no login-gated review needed but ADD the review
   account anyway under "App access": review@croftapp.co.za / CroftReview2026!
9. Promote Internal testing -> Production. First review typically 1-7 days.

## Later releases

Bump `appVersionCode` (+1) and `appVersion`/`appVersionName` in
`android/twa-manifest.json`, run `bubblewrap update` then `./build.sh`, upload
the new AAB. Needed rarely - only for icon/name/TWA-config changes; the app
content itself updates with every web deploy.
